# Database Design — Task Management System

**Database Provider:** PostgreSQL 16 (hosted on Supabase)
**ORM:** Prisma 7 with `@prisma/adapter-pg`
**Connection:** Connection pool via `pg` Pool, wired to Prisma via `PrismaPg` adapter

---

## Overview

The database uses a relational PostgreSQL schema managed through Prisma ORM. All primary keys are UUIDs generated by the database via `uuid()`. All timestamps are stored in UTC. Cascade deletes propagate from parent entities to children to maintain referential integrity automatically.

---

## Tables

### `User`

Stores all system users across all roles.

| Column | Data Type | Nullable | Default | Constraints | Description |
|---|---|---|---|---|---|
| `id` | VARCHAR (UUID) | No | `uuid()` | PRIMARY KEY | Unique user identifier |
| `name` | TEXT | No | — | NOT NULL | Full display name |
| `email` | TEXT | No | — | NOT NULL, UNIQUE | Login email |
| `password` | TEXT | No | — | NOT NULL | bcrypt-hashed password |
| `role` | ENUM | No | `COLLABORATOR` | NOT NULL | One of: `ADMINISTRATOR`, `PROJECT_MANAGER`, `COLLABORATOR` |
| `mustResetPassword` | BOOLEAN | No | `true` | NOT NULL | Requires password change on first login |
| `isActive` | BOOLEAN | No | `true` | NOT NULL | Soft deactivation; inactive users cannot log in |
| `createdAt` | TIMESTAMP | No | `now()` | NOT NULL | Record creation time (UTC) |
| `updatedAt` | TIMESTAMP | No | auto | NOT NULL | Last modification time (UTC) |

**Indexes:**
- PRIMARY KEY on `id`
- UNIQUE on `email`

### `Project`

Stores projects created and owned by Project Managers.

| Column | Data Type | Nullable | Default | Constraints | Description |
|---|---|---|---|---|---|
| `id` | VARCHAR (UUID) | No | `uuid()` | PRIMARY KEY | Unique project identifier |
| `name` | TEXT | No | — | NOT NULL | Project name (controller enforces min 3 chars) |
| `description` | TEXT | Yes | `NULL` | — | Optional project description |
| `status` | ENUM | No | `PLANNING` | NOT NULL | One of: `PLANNING`, `IN_PROGRESS`, `ACTIVE`, `COMPLETED` |
| `createdById` | VARCHAR (UUID) | No | — | NOT NULL, FK → User.id | Owner (Project Manager) |
| `createdAt` | TIMESTAMP | No | `now()` | NOT NULL | Creation timestamp |
| `updatedAt` | TIMESTAMP | No | auto | NOT NULL | Last update timestamp |

**Indexes:**
- PRIMARY KEY on `id`
- FOREIGN KEY `createdById` → `User(id)` (no cascade — user must be deleted carefully)

---

### `Task`

Stores individual work items belonging to projects.

| Column | Data Type | Nullable | Default | Constraints | Description |
|---|---|---|---|---|---|
| `id` | VARCHAR (UUID) | No | `uuid()` | PRIMARY KEY | Unique task identifier |
| `title` | TEXT | No | — | NOT NULL | Task title |
| `description` | TEXT | Yes | `NULL` | — | Optional description |
| `status` | ENUM | No | `TODO` | NOT NULL | One of: `TODO`, `IN_PROGRESS`, `COMPLETED` |
| `priority` | ENUM | No | `MEDIUM` | NOT NULL | One of: `LOW`, `MEDIUM`, `HIGH` |
| `dueDate` | TIMESTAMP | Yes | `NULL` | — | Optional due date |
| `projectId` | VARCHAR (UUID) | No | — | NOT NULL, FK → Project.id (CASCADE DELETE) | Parent project |
| `assignedUserId` | VARCHAR (UUID) | No | — | NOT NULL, FK → User.id (CASCADE DELETE) | Assigned collaborator |
| `createdAt` | TIMESTAMP | No | `now()` | NOT NULL | Creation timestamp |
| `updatedAt` | TIMESTAMP | No | auto | NOT NULL | Last update timestamp |

**Indexes:**
- PRIMARY KEY on `id`
- INDEX on `projectId` — optimizes project-scoped task queries
- INDEX on `assignedUserId` — optimizes user-scoped task queries
- FOREIGN KEY `projectId` → `Project(id)` ON DELETE CASCADE
- FOREIGN KEY `assignedUserId` → `User(id)` ON DELETE CASCADE

### `Comment`

Stores discussion comments on tasks.

| Column | Data Type | Nullable | Default | Constraints | Description |
|---|---|---|---|---|---|
| `id` | VARCHAR (UUID) | No | `uuid()` | PRIMARY KEY | Unique comment identifier |
| `content` | TEXT | No | — | NOT NULL | Comment text |
| `taskId` | VARCHAR (UUID) | No | — | NOT NULL, FK → Task.id (CASCADE DELETE) | Associated task |
| `userId` | VARCHAR (UUID) | No | — | NOT NULL, FK → User.id (CASCADE DELETE) | Author |
| `createdAt` | TIMESTAMP | No | `now()` | NOT NULL | Creation timestamp |

**Indexes:**
- PRIMARY KEY on `id`
- FOREIGN KEY `taskId` → `Task(id)` ON DELETE CASCADE
- FOREIGN KEY `userId` → `User(id)` ON DELETE CASCADE

**Access pattern:** Ordered by `createdAt DESC` in API responses.

### `Attachment`

Stores metadata for files uploaded to Supabase Storage.

| Column | Data Type | Nullable | Default | Constraints | Description |
|---|---|---|---|---|---|
| `id` | VARCHAR (UUID) | No | `uuid()` | PRIMARY KEY | Unique attachment identifier |
| `fileName` | TEXT | No | — | NOT NULL | Original filename as provided by the uploader |
| `fileUrl` | TEXT | No | — | NOT NULL | Public Supabase Storage URL |
| `taskId` | VARCHAR (UUID) | No | — | NOT NULL, FK → Task.id (CASCADE DELETE) | Associated task |
| `userId` | VARCHAR (UUID) | No | — | NOT NULL, FK → User.id (CASCADE DELETE) | Uploader |
| `createdAt` | TIMESTAMP | No | `now()` | NOT NULL | Upload timestamp |

**Indexes:**
- PRIMARY KEY on `id`
- FOREIGN KEY `taskId` → `Task(id)` ON DELETE CASCADE
- FOREIGN KEY `userId` → `User(id)` ON DELETE CASCADE

**Storage note:** The actual file is stored in Supabase Storage under the `attachments` bucket with a randomly generated hex filename. The `fileUrl` column holds the public URL to that file.

### `Notification`

Stores in-app notifications for users.

| Column | Data Type | Nullable | Default | Constraints | Description |
|---|---|---|---|---|---|
| `id` | VARCHAR (UUID) | No | `uuid()` | PRIMARY KEY | Unique notification identifier |
| `message` | TEXT | No | — | NOT NULL | Notification message text |
| `isRead` | BOOLEAN | No | `false` | NOT NULL | Read status |
| `userId` | VARCHAR (UUID) | No | — | NOT NULL, FK → User.id (CASCADE DELETE) | Recipient |

## Normalization

The database adheres to **Third Normal Form (3NF)**:

- **1NF:** All columns are atomic; no repeating groups. UUIDs are used as primary keys.
- **2NF:** All non-key attributes depend on the full primary key. No partial dependencies exist (single-column PKs throughout).
- **3NF:** No transitive dependencies. User information is not duplicated in other tables — only UUID foreign keys are stored.

## Connection Architecture

```
Backend Server
    |
    v
pg.Pool (connection pool)
    |   connectionString: DATABASE_URL (Supabase PostgreSQL)
    |   max pool size: DB_POOL_MAX (default 5)
    |
    v
PrismaPg adapter
    |
    v
PrismaClient (ORM queries)
```

The `config/prisma.js` instantiates a single shared `PrismaClient` instance using the `@prisma/adapter-pg` driver adapter, connecting through a `pg.Pool` rather than direct TCP connections. This enables Supabase connection pooling (pgBouncer) compatibility.
